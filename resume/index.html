网关服务

2018.08-2018.09

项目描述
  • 微服务化增加了模块复用性，也增加了跨部门、语言、机房、等服务调用的需求。难以解决服务发现，鉴权、服务映射，服务转发、协议转换、频控、IP白名单、调用统计、等特殊场景问题。部门服务调用具有请求量大，定制化场景多的问题，市面上的网关难以满足灵活性，并发性的需求。
  • 在这个场景下，选择进行自研网关。由于存在多种不同的网关需求，为了能快速定制化的创建部署网关服务，对网关基础功能进行抽象，变化部分做成接口，提供多种默认实现，做成Spring Boot Starter 可以快速根据场景进行网关组装与自定义实现。根据Starter，已经定制出多种场景实现，跨机房网关、业务鉴权网关、对外网关。
项目职责
  • 网关需求调研梳理，对技术进行选型，画出原型交互图。
  • 设计网关，系统流程图，与具体编码实现。
  • 推动部门同事进行网关接入，协助同事自定义网关实现
项目架构
  • 项目通过责任链模式与模板方法相结合，对请求处理进行链式处理。按照不同处理类型进行分包，每个包提供抽象接口，与默认实现。实现可插拔，可自定义实现，可以灵活组合。
  • 项目采用的是基于Netty的Vert.x Http Server异步IO框架。在偏IO处理应用上具有支持高并发，高吞吐的特性。
运用技术
  • 核心抽象统一模块，使用Spring Boot 作为脚手架，进行Starter封装。
  • 满足大流量，高并发，高IO的使用场景，使用Vert.x 异步非阻塞框架作为请求处理。
  • 验证协议提供Basic Auth 、Bearer Auth两种实现。
  • 频控提供两种基于Redis，过期自增key、相对平滑的QoS令牌桶，两种实现
  • 统计服务提供实现产生事件发送到Kafka，通过Druid进行可视化看板。
  • 转发服务默认提供基于Ribbon，根据服务名，API转发请求。
  • 业务网关请求转发增加来源标识与用户，业务服务集成配套库，通过AOP与自定义注解进行来网关源权限检验。
